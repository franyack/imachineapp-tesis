\section{[WIP]Conclusiones}
%-------------------------------------------------------------------

En este proyecto se pretendía lograr un producto de software sobre la plataforma Android que, sin la necesidad de una conexión a Internet y a partir de imágenes almacenadas en el dispositivo, sugiera una estructura de directorios para organizar el conjunto, según una medida de similitud calculada entre los elementos. Además, se necesitaba que provea funcionalidades que le permitan al usuario la interacción con el resultado, para desempeñar distintas tareas de administración sobre el almacenamiento comprendido por las imágenes. 

Un valor importante a destacar respecto al desarrollo de este trabajo es el aprendizaje de nuevas tecnologías, lenguajes de programación y arquitecturas que involucró: desde el lenguaje hasta las herramientas de software utilizadas para construir la aplicación (e.g. Java, conjunto de herramientas de desarrollo (SDK) Android), el patrón de programación necesario para que la aplicación funcione de manera correcta (MVP) y los algoritmos de extracción de características y \textit{clustering} implementados (APIs de TensorFlow, ImageNet, WordNet, MCL).

Para entender la concepción de IMachineApp como un auténtico proyecto de Ingeniería en Informática, se visualiza en la Figura \ref{fig:pie-plot-asignaturas} la contribución de cada área de asignaturas dictadas en la carrera hacia la ejecución del presente proyecto, las cuales son calculadas de forma subjetiva por el autor. Cada una de estas áreas se justifican de la siguiente forma:

\begin{enumerate}
	\item \textbf{Programación:} Para implementar cada componente del sistema, combinando distintas tecnologías y herramientas. Las asignaturas destacadas en este área son: Fundamentos de Programación, Programación Orientada a Objetos, Algoritmos y Estructuras de Datos y Tecnologías de la Programación.
	
	\item \textbf{Software:} Para gestionar el proyecto, desde el diseño hasta la ejecución, así como también aplicar buenas prácticas para el desarrollo y mantenimiento de la aplicación. Aquí se consideran las contribuciones de las siguientes asignaturas: Ingeniería del Software I y II, y Administración de Proyectos de Software.
	
	\item \textbf{Aplicación:} Para entender conceptos fundamentales en los que se basa el diseño de la biblioteca utilizada para la extracción de características de las imágenes y el algorítmo de agrupamiento utilizado, así como también saber aprovecharlos de forma correcta a la hora de definir la experimentación. Se destacan en dicha área estas materias: Inteligencia Computacional, Procesamiento Digital de Señales y Procesamiento Digital de Imágenes.
	
	\item \textbf{Matemáticas:} Para entender a bajo nivel la teoría subyacente en las funcionalidades desarrolladas, lo cual es necesario conocer para poder entender que es lo que se está implementando. Algunas de las asignaturas más importantes son: Álgebra lineal, Cálculo I, Estadística y Matemática Básica.
\end{enumerate}

Respecto a los objetivos definidos en la Sección \ref{cap1:sec:objetivos}, se considera que se pudo cumplir con lo planificado, ya que la aplicación desarrollada tiene la capacidad de sugerir una organización de directorios contenedores de imágenes similares entre sí, y a su vez brindar opciones al usuario para que pueda refinar esta sugerencia por parte del sistema según sus preferencias. Específicamente, esto se manifiesta de la siguiente manera:

\begin{itemize}
	\item Se logró implementar un motor de procesamiento de imágenes, el cual recibe un lote y las devuelve organizadas según una medida de similitud que se computa entre las mismas.
	
	\item Se definió un conjunto de imágenes a utilizar en experimentos, las cuales son lo más fiel posible a la problemática tratada en el marco de dispositivos móviles.
	
	\item Se pudo ejecutar la experimentación planteada y se logró obtener un buen desempeño respecto a métricas utilizadas en problemas de clasificación. 
	
	\item Al realizar la aplicación Android que incluye el motor de procesamiento de las imágenes, se logró que la misma sea intuitiva y fácil de utilizar por parte del usuario, permitiéndole configurar el idioma según crea conveniente, además de dar la posibilidad de personalizar el resultado obtenido según su criterio.
	
	\item La aplicación es lo suficientemente robusta como para manejar excepciones y situaciones que son consideradas fuera de lo normal, informado al usuario cual es el error e indicándole que debe realizar para continuar.
	
	\item Cumplimiento de todos los requisitos que fueron especificados en la ERS del proyecto.
\end{itemize}

\section{[WIP]Trabajos futuros}
%-------------------------------------------------------------------

A partir de todo el trabajo realizado, también se concluye que el software desarrollado tiene diversos aspectos a perfeccionar en términos de desempeño y utilidad, los cuales fueron presentados en la Sección \ref{cap1:subsec:evolucion}. Específicamente, se consideran las siguientes cuestiones a mejorar en un futuro:

\begin{itemize}
	\item Incorporación de nuevas características que ayuden a describir mejor las imágenes, con el objetivo de obtener mejores resultados.
	
	\item En base a lo realizado en la Sección \ref{cap2:sec:experimentos}, se concluye que los tiempos de proceso para celulares de gama baja/media representan un problema importante, por lo que se deberían estudiar nuevas formas o metodologías para poder aumentar el número de imágenes a procesar a la vez.
	
	\item Reducir el consumo de almacenamiento durante la etapa del proceso, guardando las rutas de las imágenes afectadas y no haciendo una copia de las mismas como hace la aplicación hasta el momento.
	
	\item Darle la posibilidad al usuario de que configure ciertos parámetros con el objetivo de ``jugar'' con la aplicación y hacer más personalizada la sugerencia de organización de las imágenes.
	
	\item Ofrecer la posibilidad al usuario de separar y diferenciar aquellas imágenes catalogadas como ``memes'', con el objetivo de que sea más fácil identificarlas y borrarlas.
	
	\item Darle un nombre representativo a cada carpeta, basándose en aquel \textit{tag} que más ocurrencias presente al formar la matriz de afinidad gramatical.
	
	\item Implementar un proceso de integración continua para mejorar los mecanismos de pruebas y despliegue de la aplicación.
	
	\item Que el motor de procesamiento de las imágenes pueda ser utilizado en otro ambiente, como por ejemplo otro sistema operativo \textit{mobile}, una aplicación \textit{desktop}, o bien como un servicio alojado en la nube y consumido a través de diferentes proyectos web.
\end{itemize}

Dado que las tecnologías comprendidas por la aplicación se siguen desarrollando y mejorando cada vez más, logrando con ello diversas aplicaciones de forma exitosa, se puede concluir finalmente que IMachineApp provee las bases que necesita una aplicación Android para realizar procesamiento y agrupación sobre un conjunto de imágenes, y puede seguir mejorando sus capacidades para ser una herramienta flexible de personalizar y simple de utilizar en tareas complejas.
